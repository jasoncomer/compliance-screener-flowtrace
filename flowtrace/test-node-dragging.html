Technical Details of the Fix

Root Cause

The issue was caused by setting both draggedNode (for node dragging) and isDragging (for canvas panning) simultaneously when a node was clicked, creating a conflict between the two dragging modes.

Solution

Separated the concerns by:

  • Node Dragging: Only set draggedNode when clicking on nodes (not isDragging)
  • Canvas Panning: Only set isDragging when clicking on empty space (not draggedNode)
  • Event Handling: Mouse move handler checks these states separately and handles them independently

Code Changes

Modified components/network-graph.tsx:

  • Removed setIsDragging(true) from node click handler
  • Maintained coordinate transformation logic for proper zoom handling
  • Ensured pan update logic only triggers for canvas dragging
