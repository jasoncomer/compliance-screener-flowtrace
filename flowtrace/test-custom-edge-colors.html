Feature Added: Users can now set custom colors for individual transaction edges in the FlowTrace graph.

How to Use Custom Edge Colors

1. Load the Application

  • Open the FlowTrace application
  • Verify the graph loads with nodes and connections visible

2. Access Edge Details

  • Click on any single connection/edge between nodes
  • A popup dialog will appear showing transaction details
  • Look for the "Customize Edge Color" button

3. Set Custom Color

  • Click the "Customize Edge Color" button
  • A color picker dialog will open with 12 predefined colors
  • Click on any color to apply it to the edge
  • The edge will immediately update with the new color

4. Available Colors

  • Gray (Default)
  • Purple
  • Cyan
  • Green
  • Yellow
  • Red
  • Lime
  • Pink
  • Orange
  • Indigo
  • Violet
  • Sky

5. Reset to Default

  • In the color picker dialog, click "Reset to Default"
  • The edge will return to the default gray color
  • This removes the custom color setting

6. Test Multiple Edges

  • Try setting different colors for multiple edges
  • Verify that each edge maintains its own color independently
  • Test with both single connections and multiple connections between the same nodes
Expected Result: Individual edges can be colored independently, making it easier to distinguish between different transactions and highlight important connections.
Note: Custom colors are applied to individual connections using their transaction hash (txHash) as a unique identifier. Colors persist until manually reset or the connection is modified.

Technical Implementation

Data Structure Changes

The Connection interface now includes an optional customColor property:

interface Connection {
  // ... existing properties
  customColor?: string // Custom color for individual edges
}

Rendering Logic

The rendering logic prioritizes custom colors in this order:

  • Individual Connection: connection.customColor
  • Group Custom Colors: customEdgeColors[connectionKey]
  • Default: Gray color based on theme

Utility Functions

Three utility functions are available for managing custom colors:

  • setConnectionCustomColor(txHash, color) - Set custom color for a connection
  • clearConnectionCustomColor(txHash) - Remove custom color from a connection
  • getConnectionCustomColor(txHash) - Get current custom color for a connection

User Interface

The feature is accessible through:

  • Click on any single connection to open edge details
  • "Customize Edge Color" button in the edge details dialog
  • Color picker with 12 predefined colors
  • "Reset to Default" option to remove custom colors
